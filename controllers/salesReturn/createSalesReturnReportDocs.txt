PURPOSE: Step-by-step documentation of the "createSalesReturn" API function.

FUNCTION NAME:

createSalesReturn

DESCRIPTION:

This API endpoint creates a sales return record for goods returned by retailers. It validates all related entities, checks bill creation date constraints, manages RBP (Reward Bonus Points) validations, updates inventory levels, creates credit notes or replacements based on return type, processes point transactions, and maintains complete audit trails across multiple models.

The function ensures data consistency between SalesReturn, Bill, Inventory, CreditNote, Replacement, Transaction, Ledger, DistributorTransaction, and RetailerMultiplierTransaction models.

STEP 1: Extract and Validate Required Fields

Extract the following required fields from req.body:
{
  billId,
  salesmanName,
  routeId,
  retailerId,
  goodsType,
  collectionStatus,
  remarks,
  lineItems,
  totalBasePoints,
  grossAmount,
  schemeDiscount,
  distributorDiscount,
  taxableAmount,
  cgst,
  sgst,
  igst,
  invoiceAmount,
  roundOffAmount,
  cashDiscount,
  netAmount
}

Field Validation:

- Check for missing required fields: billId, salesmanName, routeId, retailerId, goodsType

- If any missing → HTTP 400 Bad Request

→ Error: "Missing required fields: [field names]"

STEP 2: Validate Related Entities Existence


Perform parallel validation using Promise.all():


1. 
Bill Validation:


	- Bill.findById(billId)

	- If not found → HTTP 404 with message "Bill not found"


2. 
Salesman Validation:


	- Employee.findById(salesmanName)

	- If not found → HTTP 404 with message "Salesman not found"


3. 
Beat Validation:


	- Beat.findById(routeId)

	- If not found → HTTP 404 with message "Beat not found"


4. 
Retailer Validation:


	- OutletApproved.findById(retailerId)

	- If not found → HTTP 404 with message "Retailer not found"


5. 
Distributor Validation:


	- Distributor.findById(req.user._id).lean()

	- If not found → HTTP 404 with message "Distributor not found"

    STEP 3: Generate Sales Return Number and Validate Bill Date

1. 


Generate Sales Return Number:


	- const salesReturnNo = await generateCode("SR")


2. 
Bill Date Validation:

	- Extract bill creation date using moment.tz(bill.createdAt, "Asia/Kolkata")

	- Calculate difference with current date: currentDate.diff(billCreationDate, "days")

	- If difference > 90 days → HTTP 400 Bad Request

→ Error: "Bill Creation date should be within 90 days from today"

STEP 4: RBP Scheme Validation and Point Balance Check

1. 


Initialize Store Points:


	- let storePoints = 0


2. 
Point Balance Check (Conditional):

	- Condition: totalBasePoints > 0 AND distributor.RBPSchemeMapped === "yes"

	- Fetch retailer's current point balance:

const res = await axios.get(`${RBP_POINT_BALANCE_CHECK_RETAILER}?id=${retailerId}`);
storePoints = Number(res.data?.balance) || 0;

If API call fails → HTTP 400 with message "Error fetching store points balance"

Calculate Multiplier Points to Deduct:

- Condition: Only if distributor.RBPSchemeMapped === "yes"

- Call helper function: getMultiplierPointsToDeduct(bill, totalBasePoints)

- Extract totalMultiplierPointsToDeduct from result

1. 

Total Points Validation:


	- Condition: RBP scheme mapped AND totalBasePoints > 0

	- Required points = totalBasePoints + totalMultiplierPointsToDeduct

	- If storePoints < requiredPoints → HTTP 400 Bad Request

→ Error: "Insufficient store points. Available: X, Required: Y"

Initialize Arrays:

For Each Line Item:

1. 
Inventory Validation:


	- Inventory.findById(item.inventoryId)

	- If not found → HTTP 404 with message "Inventory item not found: ${inventoryId}"


2. 
Product and Price Retrieval:


	- Product.findById(item.product)

	- Price.findById(item.price)

	- Calculate pieces per box and unit prices


3. 
Inventory Updates Based on Goods Type:


For Salable Goods:

inventory.availableQty += Number(item.returnQty);
inventory.totalStockamtDlp += Math.round(dlpbyPcs * Number(item.returnQty));
inventory.totalStockamtRlp += Math.round(rlpbyPcs * Number(item.returnQty));

For Unsalable Goods:
inventory.unsalableQty += Number(item.returnQty);

Update Total Quantity:
inventory.totalQty = (inventory.availableQty || 0) + (inventory.unsalableQty || 0);
await inventory.save();

Create Transaction Record:
const transaction = await Transaction.create({
  distributorId: req.user._id,
  transactionId: stockId,

Categorize Items by Return Type:



- If item.salesReturnType === "Credit Note" → Add to creditNoteItems

- If item.salesReturnType === "Replacement" → Add to replacementItems

STEP 7: Create Credit Note (If Applicable)


Condition: If creditNoteItems.length > 0


1. 
Calculate Total Amount:
const totalAmount = creditNoteItems.reduce(

const creditNoteNo = await generateCode("CN");

- Fetch latest ledger balance

- Generate transaction ID: ledgerTransactionCode("LEDG", req.user._id)

- Create ledger entry with debit transaction for credit note amount

Condition: totalBasePoints > 0 AND distributor.RBPSchemeMapped === "yes"

1. 
Get Current Distributor Balance:

const data = {

STEP 11: Helper Function - getMultiplierPointsToDeduct


Purpose: Calculate multiplier points to deduct based on historical multiplier transactions

Logic:


1. Check Current Month: If bill is from current month → no deduction

2. Scan Previous 3 Months: Find highest multiplier percentage earned

3. Calculate Points: Math.round((totalBasePoints * highestPercentage) / 100)

Return: Object with pointsToDeduct and percentage properties
