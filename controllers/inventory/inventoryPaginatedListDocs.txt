FUNCTION NAME:


inventoryPaginatedList

DESCRIPTION:


This API retrieves a paginated list of inventory items for a distributor with comprehensive filtering, searching, and stock calculation capabilities. It provides detailed inventory information including product details, stock quantities, in-transit quantities, closing stock calculations, and optional RBP (Reward-Based Program) points calculation for distributors enrolled in the RBP scheme.

The function is designed for performance and scalability with:


- MongoDB aggregation pipelines for efficient data retrieval

- Flexible filtering options for products and stock types

- Pagination support for large datasets

- Real-time stock calculations including in-transit quantities

- Historical closing stock calculations based on transaction history

- RBP points calculation for eligible distributors

STEP 8: Calculate Closing Stock (if requested)

if (closingStockDate && stockType) {
  let endDate = new Date(closingStockDate);
  endDate.setHours(23, 59, 59, 999);

  inventories = await Promise.all(
    inventories.map(async (invItem) => {
      const transactions = await Transaction.find({
        $and: [
          { distributorId: distributorId },
          { productId: invItem.productId },
          { createdAt: { $lt: endDate } },
          { stockType: stockType },
        ],
      }).sort({ createdAt: -1 });

      if (transactions.length > 0) {
        const lastTransaction = transactions[0];
        return {
          ...invItem,
          closingStockCount: lastTransaction?.balanceCount,
        };
      } else {
        return {
          ...invItem,
          closingStockCount: null,
        };
      }
    })
  );
}

Purpose:


- Calculates historical closing stock for a specific date if requested

- Sets end date to end of day (23:59:59.999) for accurate date comparison

- Finds the last transaction before the closing date for each product

- Uses the balance count from the last transaction as closing stock

STEP 10: Calculate RBP Points (for eligible distributors)

const distributor = await Distributor.findById(distributorId).select("RBPSchemeMapped");

let currentStockTotalPoints = null;

if (distributor?.RBPSchemeMapped === "yes" && stockType === "salable") {
  const pointsCalculationPipeline = [
    {
      $match: {
        distributorId: distributorId,
        $or: [{ availableQty: { $gt: 0 } }, { reservedQty: { $gt: 0 } }],
      },
    },
    {
      $lookup: {
        from: "products",
        localField: "productId",
        foreignField: "_id",
        as: "product",
      },
    },
    { $unwind: "$product" },
  ];

  const allInventoriesForPoints = await Inventory.aggregate(pointsCalculationPipeline);

  currentStockTotalPoints = allInventoriesForPoints.reduce((totalPoints, invItem) => {
    const basePoint = parseFloat(invItem.product?.base_point) || 0;
    const availableQty = Number(invItem.availableQty) || 0;
    const reservedQty = Number(invItem.reservedQty) || 0;
    const totalQty = availableQty + reservedQty;
    const productPoints = basePoint * totalQty;
    return totalPoints + productPoints;
  }, 0);
}

Purpose:


- Checks if distributor is enrolled in RBP scheme

- Calculates total points for salable stock only

- Multiplies base points per product by total available quantities

- Includes points calculation in pagination response

DATA MODELS INVOLVED

Collections Used:

1. Inventory - Main inventory records

2. Products - Product master data

3. Distributors - Distributor information

4. Transactions - Stock transaction history

5. Invoices - Invoice data for in-transit calculations

Key Fields:

- Inventory: availableQty, reservedQty, unsalableQty, offerQty, godownType

- Products: name, product_code, brand, cat_id, collection_id, base_point

- Distributors: RBPSchemeMapped

- Transactions: balanceCount, stockType, createdAt

- Invoices: status, lineItems

BUSINESS RULES

1. Authentication: Only authenticated distributors can access their own inventory

2. Stock Types: Three types supported - salable, unsalable, offer

3. Zero Stock: Can be included or excluded based on showZeroStock parameter

4. RBP Points: Only calculated for distributors with RBPSchemeMapped = "yes"

5. Closing Stock: Calculated from transaction history up to specified date

6. In-Transit: Calculated from invoices with "In-Transit" status

7. Search: Case-insensitive search across product code and name

8. Pagination: Default 10 items per page, maximum controlled by limit parameter

PERFORMANCE CONSIDERATIONS

1. Aggregation Pipeline: Uses efficient MongoDB aggregation for joining collections

2. Parallel Execution: Runs multiple queries simultaneously using Promise.all

3. Indexed Queries: Relies on proper indexing of distributorId, productId, and status fields

4. Selective Fields: Uses .select() and .lean() for better performance

5. Batch Processing: Handles large datasets through pagination

ERROR HANDLING

1. Try-Catch: Wrapped in asyncHandler for consistent error handling

2. Validation: Checks for required fields and proper data types

3. Fallbacks: Provides default values for optional parameters

4. Error Response: Returns standardized error format with status codes

DEPENDENCIES

1. express-async-handler: For async error handling

2. mongoose: For MongoDB operations and ObjectId conversions

3. Models: Inventory, Transaction, Invoice, Distributor models
