PURPOSE: Step-by-step documentation of the "ProcessRetailerMultiplierTransaction" API function.

FUNCTION NAME:

ProcessRetailerMultiplierTransaction

DESCRIPTION:

This API endpoint processes monthly retailer multiplier transactions based on various performance metrics. It calculates and awards Volume Multiplier, Consistency Multiplier, and Bill Volume Multiplier points to retailers based on their sales performance, bill consistency, and transaction volumes for a specific month/year combination.

The function integrates with external RBP (Reward Bonus Points) APIs to credit calculated multiplier points to retailers' accounts while maintaining complete audit trails and preventing duplicate processing.

The function ensures data consistency between RetailerMultiplierTransaction, Bill, DistributorTransaction, OutletApproved models and external RBP systems.

STEP 7: Calculate Volume Multiplier Points

Volume Multiplier Logic:
let volumeMultiplierPoint = 0;
let volumeMultiplierPercentage = 0;

if (
  volumeMultiplierSlab &&
  volumeMultiplierSlab.status?.toLowerCase() === "active" &&
  totalPoints > 0
) {
  const applicableSlab = [...volumeMultiplierSlab.slabs]
    .sort((a, b) => Number(b.slabName) - Number(a.slabName))
    .find((slab) => Number(totalPoints) >= Number(slab.slabName));

  volumeMultiplierPercentage = applicableSlab?.percentage || 0;

  if (applicableSlab) {
    volumeMultiplierPoint = Math.round(
      (Number(applicableSlab.percentage) * totalPoints) / 100
    );
  }
}

Algorithm:

1. Check if Volume Multiplier slab is active

2. Ensure totalPoints > 0

3. Sort slabs in descending order by threshold

4. Find the highest slab that the retailer qualifies for

5. Calculate multiplier points: (percentage * totalPoints) / 100

STEP 10: Helper Function - getRetailersBillMonthConsistency


Consistency Calculation Logic:
const getRetailersBillMonthConsistency = async (retailerId, month, year) => {
  try {
    let consistency = 0;
    for (let i = 0; i < 3; i++) {
      const start = moment
        .tz({ year, month: month - 1 - i, day: 1 }, "Asia/Kolkata")
        .startOf("day")
        .toDate();
      const end = moment(start).endOf("month").toDate();

      const billCount = await Bill.countDocuments({
        retailerId,
        status: "Delivered",
        "dates.deliveryDate": { $gte: start, $lt: end },
      });

      if (billCount > 0) {
        consistency++;
      } else {
        break; // Break on first month without bills
      }
    }
    return consistency;
  } catch (error) {
    console.error("Error in getRetailersBillMonthConsistency:", error.message);
    return 0;
  }
};

Algorithm:

1. Check last 3 months (including current processing month)

2. Count delivered bills for each month

3. Increment consistency counter for months with bills

4. Break on first month without bills (consistency must be consecutive)

5. Return total consecutive months with bills

    amount: volumeMultiplierPoint,
    remarks: `Volume Multiplier for ${monthName}, ${year} based total points ${totalPoints}`,
    type: "Sales Multiplier",
  };

  try {
    const earnPointsResponse = await axios.post(RBP_POINT_CREDIT_API, body);
    if (earnPointsResponse.data?.error) {
      data.status = "Failed";
      data.apiResponse = earnPointsResponse.data;
    } else {
      data.status = "Success";
    }

  transactionsToSave.push(new RetailerMultiplierTransaction(data));
}

Process for Each Multiplier Type:


1. Volume Multiplier - Based on total points earned

2. Consistency Multiplier - Based on consecutive months with bills + total points

3. Bill Volume Multiplier - Based on total bill amount threshold + total points

External API Integration:


- Each multiplier type calls external RBP API to credit points

- API response determines transaction status (Success/Failed)

- Failed API calls are logged but don't halt processing
